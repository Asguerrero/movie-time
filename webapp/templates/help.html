REQUEST: /movies

This endpoint is for the list of filtered movies.

GET parameters:

The results are constrained depending on the following optional GET parameters, which will be combined by a boolean AND:

  1. title (Optional, default: ' ')
  2. budget (Optional, default: ' ')
  3. languages (Optional, default: ' ')
  4. rating (Optional, default: ' ')
  5. company (Optional, default: ' ')
  6. countries (Optional, default: ' ')
  7. years (Optional, default: ' ')
  8. revenue (Optional, default: ' ')
  9. genres (Optional, default: ' ')
  10. runtime (Optional, default: ' ')

RESPONSE: A JSON list of dictionaries, each of which represents one movie, sorted by descending order of release date. Each book dictionary will have the following fields:

  1. movie_id -- (integer)
  2. title -- (string) // the title of the movie
  3. poster_path -- (string) // the image pathname of the movie's poster
  4. year -- (integer) // the release year, represented as a 4-digit integer
  5. genre -- (list) // the genre(s) of the movie
  6. rating -- (float) // the movie's rating on a scale from 1-10

EXAMPLE(S):

api/movies?genre=Family

[{"id": 26555, "title": "Avatar 2", "release_year": 2020, "rating": 0.0}, {"id": 30394, "title": "Bad Boys for Life", "release_year": 2018, "rating": 0.0}, {"id": 38873, "title": "The Other Side of the Wind", "release_year": 2018, "rating": 0.0}, {"id": 33351, "title": "Iron Sky: The Coming Race", "release_year": 2018, "rating": 0.0}...]

REQUEST: /movie/{movie_id}

This endpoint is for an specific movie.

GET parameters

  1. movie_id (Mandatory)

REPONSE: A JSON dictionary that represents a specific movie with the following fields:

  1. title -- (string) // the title of the movie
  2. poster_path -- (string) // the image pathname of the movie's poster
  3. year -- (integer) // the release year, represented as a 4-digit integer
  4. genre -- (list) // the genre(s) of the movie
  5. rating -- (float) // the movie's rating on a scale from 1-10
  6. overview -- (string) // a short description of the plot of the movie
  7. runtime -- (integer) // duration of the movie in minutes
  8. budget -- (float) // budget of the movie
  9. revenue -- (float) // total box office profit the movie made
  10. IMBD_id -- (integer) // the title ID that can be used to search the title on IMDb
  11. language -- (string) // the main language spoken
  12. spoken_languages -- (list) // the spoken language(s) in the movie
  13. prod_companies -- (list) // the companies that were involved in the movie's production
  14. prod_countries -- (list) // the countries in which the movie was produced

EXAMPLE(S):

api/movie/123

{"id": 123, "title": "Flirting with Disaster", "overview": "Adopted as a child, new father Mel Colpin (Ben Stiller) decides he cannot name his son until he knows his birth parents, and determines to make a cross-country quest to find them. REQUEST: /genres

RESPONSE: a JSON list of strings sorted in alphabetical order.

EXAMPLE(S):

api/genres

["Animation", "Comedy", "Family", "Adventure", ...]

REQUEST: /languages

RESPONSE: A JSON list of strings sorted in alphabetical order (put the 5 most spoken languages at the top and the rest ordered). api/languages

EXAMPLE(S):

api/languages

["English", "Fran\u00e7ais", "Espa\u00f1ol", "Deutsch" ...]

I know it looks strange here, but trust me, it renders correctly with javascript.

REQUEST: /countries

RESPONSE: A JSON list of countries sorted in alphabetical order.

EXAMPLE(S):

api/countries

["United States of America", "Germany", "United Kingdom" ...]
